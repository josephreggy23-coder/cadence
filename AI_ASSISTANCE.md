# AI assistance and authorship

CADENCE has used AI-assisted software development and research auditing. AI tools
have helped inspect repositories, identify methodological risks, implement and
test code, locate public datasets, and edit technical documentation. The Git
history preserves that development trail.

This disclosure matters because the AAN Neuroscience Research Prize requires the
research and written application to be the applicant's original work. The public
rules do not currently explain how AI-assisted coding is evaluated.

Before treating CADENCE as a competition project, the student should:

1. obtain written guidance from AAN about permitted AI assistance and disclosure;
2. maintain a dated log of student decisions, experiments, debugging, and mentor
   input;
3. independently reproduce and understand every submitted result;
4. write the competition abstract and report personally rather than submitting
   repository prose as original student writing;
5. disclose the public source datasets and the contributions of their authors;
6. keep exploratory results separate from later confirmatory work.

No repository badge, commit count, or generated figure substitutes for original
student scientific judgment. If AAN says the existing assistance is outside its
rules, this repository can remain a learning project but should not be submitted
as an eligible individual entry.
